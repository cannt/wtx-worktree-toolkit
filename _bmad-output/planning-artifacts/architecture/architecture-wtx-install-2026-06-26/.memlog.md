- (constraint) bash 3.2 compat: no declare -A, no readarray, no readlink -f, no process substitution to array — confirmed in SPEC constraints and existing codebase
- (constraint) set -u only, never set -e — enforced across all existing scripts; new scripts must follow the same convention
- (constraint) no eval of raw user input anywhere — existing lib guards this; installer must maintain it
- (constraint) all config reads via wtx_config_get / wtx_config_get_list — no ad-hoc TOML parsing in new code
- (constraint) gum wrappers with pure-bash read fallback on every prompt — tui_* functions in lib/worktree-tui.sh cover this; wizard sources that lib
- (constraint) no hardcoded org/project/Jira/forge literals — all values from user input or config layer at runtime
- (constraint) wtx init remains independently usable; wtx install does not deprecate or remove it
- (constraint) install.sh remains canonical authority for symlink creation, hooks copy, gradle init — wizard delegates, does not reimplement
- (constraint) no new config schema fields: wizard configures the existing wtx.example.toml schema only; AD-9 locks this
- (decision) AD-1 ROUTING: wtx install dispatches via bin/wtx case statement to scripts/worktree-install.sh, mirroring the start/done/status pattern. Prevents a standalone invocation path that skips WTX_ROOT/WORKSPACE_ROOT resolution. [ADOPTED — consistent with existing dispatcher]
- (decision) AD-2 FILE LAYOUT: wizard logic in scripts/worktree-install.sh; TOML write helpers (_wtx_toml_escape, _wtx_csv_to_toml_array) and plugin-discovery (wtx_install_discover_plugins) moved to lib/wtx-install.sh; bin/wtx sources lib/wtx-install.sh before calling _wtx_init. Prevents helpers duplicated across bin/wtx and the new script, and prevents the install script reimplementing TOML escaping independently.
- (decision) AD-3 DELEGATION: symlink creation, hooks copy, and Gradle init are delegated to install.sh via bash subprocess (bash "$WTX_ROOT/install.sh" --prefix ...), not reimplemented. Prevents two implementations of the same file operations diverging. Dry-run flag is forwarded as --dry-run to the subprocess.
- (decision) AD-4 ATOMIC TOML WRITE: wtx.toml is written to a mktemp file in WORKSPACE_ROOT then mv'd into place. A trap on EXIT removes the temp file. Prevents a partially written wtx.toml on Ctrl-C mid-write. Applies to new write and merge paths; skip path touches nothing.
- (decision) AD-5 DRY-RUN FLAG: a session-scope variable WTX_INSTALL_DRY_RUN (0|1) set at preflight is exported and checked at every write site. All install.sh subprocess calls append --dry-run when set. write_or_dryrun() wrapper function in worktree-install.sh encapsulates this. Prevents dry-run code paths scattered with ad-hoc flag checks.
- (decision) AD-6 MERGE PRE-FILL: on 'merge' path, wizard pre-fills each tui_input default by calling wtx_config_get against the existing wtx.toml (WTX_CONFIG set to that path before sourcing lib/wtx-config.sh). New file is written via AD-4 atomic write, not in-place edit. Prevents in-place patch strategies that corrupt the existing file on partial runs. [ADOPTED — locked by user as given decision]
- (decision) AD-7 SUMMARY LEDGER: a summary associative structure is simulated with positional arrays (_LEDGER_KEYS and _LEDGER_VALS parallel arrays, bash 3.2 safe) accumulating status:label pairs. Step 11 iterates both to render the completion table. Prevents each step independently managing its own display state.
- (decision) AD-8 PLUGIN DISCOVERY: wtx_install_discover_plugins() in lib/wtx-install.sh scans $WTX_ROOT/plugins/*.sh, reads the first line matching '^# wtx-plugin-desc:' comment header, returns filename TAB description pairs via stdout (newline-separated). 'None' and 'Custom path...' are prepended by the wizard, not by the discovery function. Prevents a manifest file that could drift from actual plugin files. [ADOPTED — locked by user as given decision]
- (decision) AD-9 NO SCHEMA EXTENSION: wizard configures only fields present in wtx.example.toml as of the feature spec; no new TOML keys are introduced. Prevents wtx_config_get callers encountering keys the loader doesn't model. [ADOPTED — from SPEC non-goals]
- (decision) AD-10 TUI CONSISTENCY: all interactive prompts in worktree-install.sh use tui_input / tui_confirm / tui_choose / tui_style_box from lib/worktree-tui.sh (sourced with inline stub fallback block). Prevents ad-hoc read prompts bypassing the gum/bash toggle and breaking Step 0 gum-detection flag.
- (decision) AD-11 PATH HINT GATE: PATH hint in Step 10 is shown only when $PREFIX/bin is NOT in $PATH. Detection: case ":$PATH:" in *":$PREFIX/bin:"*). [ADOPTED — locked by user as given decision]
- (decision) AD-12 PREFLIGHT ORDER: Step 0 runs three checks in fixed order — (1) git repo presence, (2) gum detection into GUM_AVAILABLE variable, (3) --dry-run flag parse — before any user prompt. The wizard exits non-zero on (1) fail; (2) and (3) only set session flags. This order is binding so the TTY-require check runs before any tui_ call.
- (decision) AD-13 IDEMPOTENCY GATE: when WORKSPACE_ROOT/wtx.toml exists, wizard prompts skip/overwrite/merge before any other step (after Step 0 preflight, before Step 1 banner). On 'skip', wizard jumps to hooks/extras/summary. On 'overwrite', clears prefill, runs full config flow. On 'merge', loads prefill via AD-6. This gate is the single branch point for CAP-5 — not repeated at individual steps.
