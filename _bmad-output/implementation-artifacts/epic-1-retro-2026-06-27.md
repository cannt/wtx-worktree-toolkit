## 3. What Went Well

1. **The Story 1.1 foundation paid compounding dividends.** A single library
   (`lib/wtx-install.sh`) owned every cross-cutting primitive — `_wtx_toml_escape`,
   `_wtx_csv_to_toml_array`, `wtx_install_write_or_dryrun`, plugin discovery, and the
   `_WTX_LEDGER_KEYS`/`_WTX_LEDGER_VALS` arrays. Stories 1.2–1.7 consumed these without
   re-implementing them. The architecture spine (AD-1..AD-13) worked as designed.

2. **Single write chokepoint validated by dry-run.** Routing every mutation through
   `wtx_install_write_or_dryrun` (AD-5) made Story 1.6's end-to-end dry-run threading
   nearly mechanical — no scattered `--dry-run` checks. A design decision from Story 1.1
   removed an entire class of work three stories later.

3. **Monotonic regression discipline.** Test count never regressed: 53 → 327. Each story
   added focused coverage plus reviewer-added gap cases, and all six suites stayed green
   at every handoff.

4. **bash 3.2 constraint held end-to-end.** Parallel indexed arrays for the ledger, no
   `declare -A`, no `mapfile`/`readarray`, no `readlink -f`. NFR1 never slipped.

5. **The dev → review → auto-fix loop caught real defects every time.** The Senior Review
   (Codex / GPT-5) surfaced genuine HIGH-severity bugs in 1.1, 1.2, 1.3, and 1.5 — all
   fixed before the story closed.

## 4. Challenges & Recurring Patterns

> No blame — these are systemic patterns to address with process, not people.

### Pattern A — Failure propagation / return-code correctness (HIGH-impact, ≥2 stories)
- **1.2 (2× HIGH):** `_wtx_install_step2_binary` recorded `symlink=failed` in the ledger
  but returned `0`, so the wizard could exit success despite a failed `install.sh`
  delegation. Separately, TOML-write failures were swallowed with no `failed` ledger entry.
- Threaded forward as "optional-rc" handling that had to be re-verified in 1.3, 1.4, 1.7.
- **Root cause:** in `set -u`-only / no-`set -e` bash, a function's exit status must be
  *explicitly* propagated; the ledger and the return code were maintained as two separate
  truths that drifted.

### Pattern B — Working-directory & path-resolution correctness (≥2 stories)
- **1.1 (HIGH):** preflight resolved `WORKSPACE_ROOT` before parsing `--dry-run`, breaking
  Step 0 order; and the direct-invocation `WTX_ROOT` fallback didn't match the installed
  layout.
- **1.3 (HIGH):** `install.sh --hooks` copies to `$PWD/.claude/hooks`; invoking
  `wtx install` from a nested directory satisfied the ledger while leaving
  `$WORKSPACE_ROOT/.claude/hooks/` untouched. Fixed by `cd "$WORKSPACE_ROOT"` around the
  delegated write.
- **Root cause:** subprocess delegation to `install.sh` implicitly depends on cwd.

### Pattern C — No side effects before user consent (1 story, HIGH)
- **1.5 (HIGH):** preflight created `.wtx-install-tmp.*` before the idempotency
  skip/overwrite/merge choice, violating AC-1's "no files touched before choice." Fixed by
  deferring temp allocation to commit time.

### Pattern D — Commit / File-List hygiene (MOST frequent — 4 of 7 stories)
- **1.4, 1.5, 1.6, 1.7** each had review findings for uncommitted test artifacts
  (`tests/test-summary.md`, `_bmad-output/story-automator/orchestration-*.md`) or File List
  entries that didn't match the actual git diff.
- **Root cause:** automation-generated side artifacts (test summaries, orchestration
  progress) were not part of the dev's File List discipline.

### Pattern E — QA coverage gaps reaching review (4 stories)
- Reviewers added gap-coverage cases in 1.2, 1.4, 1.5, 1.7 (e.g., ledger assertions for
  already-on-PATH, merge re-source, custom marker/hook paths, no-gum fallback rendering).
- The loop caught them — but the same defect *classes* recurred at the review boundary
  rather than being caught at dev time.

---

## 5. Key Insights

1. In `set -u`/no-`set -e` bash, **ledger state and function return code are two truths
   that must be written together** or they drift (Pattern A).
2. **Any delegation to `install.sh` is cwd-sensitive** and must pin `WORKSPACE_ROOT`
   explicitly (Pattern B).
3. **Consent/commit gates must own all side-effect allocation** — including temp files
   (Pattern C).
4. **Automation side-artifacts need to be in the File List contract**, or every story pays
   a review-cycle tax (Pattern D).
5. The **architecture-spine-first sequencing** (build all primitives in Story 1.1) is the
   single biggest reason velocity stayed high — worth repeating as a deliberate pattern.
